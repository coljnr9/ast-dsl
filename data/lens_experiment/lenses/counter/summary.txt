The CTU (Count Up) system is a specialized function block designed for industrial automation tasks that require the tracking and incrementing of discrete events. Its primary purpose is to maintain a running tally of signal transitions and provide status updates when a predefined numerical threshold is reached. By monitoring a specific input signal, the system allows processes to trigger subsequent logic or alarms once a designated count has been achieved, making it a fundamental component for batch processing and inventory management within a controller application.

The operation of the system centers on the relationship between three input parameters and two output variables. The core entity is the Current Value (CV), a word-based integer that represents the internal state of the counter. The system increments this value by one every time it detects a rising edge on the Count Up (CU) input, which signifies a transition from a low to a high state. Simultaneously, the system monitors the Preset Value (PV), which serves as the target limit. The relationship between these values determines the state of the output signal (Q); specifically, the system sets the output to true whenever the current value is greater than or equal to the preset value. This creates a logical bridge between the numerical tally and the binary state of the overall system.

The behavior of the CTU is governed by several critical constraints and priority rules. The reset functionality takes precedence over the incrementing logic; if the reset input is activated, the current value is immediately forced to zero regardless of any activity on the count input. This reset action also serves to deactivate the output signal, as the current value will fall below the preset value. It is important to note that the system relies exclusively on rising edges for its counting operations, meaning a sustained high signal on the input will not result in repeated increments. Additionally, because the current value is stored as a word data type, the system is constrained by the maximum numerical capacity of that format, and the logic for the output is inclusive of the preset value itself, ensuring the signal triggers the moment the goal is met.
