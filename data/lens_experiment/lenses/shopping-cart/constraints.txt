1. Only visitors to the web server can select items for eventual purchase.
2. Customers can accumulate a list of items for purchase over time.
3. The software must allow the user to perform a final review of the shortlisted purchase before the point of sale.
4. Users must be permitted to modify their purchase selection during the review process.
5. The software calculates the order total by summing the product prices, freight transport, postage, packaging, and labeling.
6. Applicable taxes must be calculated and added to the order total at the point of sale.
7. Credit card transactions must be conducted through a secure payment gateway rather than the software's core module.
8. Sensitive payment information must be captured via the secure gateway provider's software module.
9. Shopping cart data identifies the user's session using HTTP cookies or query strings.
10. Temporary cart data must be stored in a session object and manipulated in real time as items are selected.
11. Finalizing a transaction triggers the generation of an order against the selected items.
12. The shopping cart must be cleared immediately after an order is successfully generated.
13. Storefront pages, including category and product views, are dynamically generated based on database information.
14. Merchants can manage the online shop through a dedicated administration area.
15. A merchant can add, edit, or delete products, categories, and discounts via the administration interface.
16. Merchants can configure shipping and payment settings within the administration area.
17. If an API is provided, third-party solutions may export order data to external systems like accountancy platforms.
18. Open-source software users are permitted to access and modify the entire source code of the online store.
19. Licensed software users must install and host the application on a web server that meets specific server requirements.
20. Hosted service providers may charge a percentage of sales in addition to recurring subscription fees.
21. Users of hosted services are restricted to customizing the site within the limits of predefined templates or specific templating languages.
22. Hosted service vendors are responsible for the continuous deployment of security patches and new features.
23. The software must facilitate the storage of product, category, discount, order, and customer data in a database.
24. Payment account data must be stored, transmitted, and handled according to PCI security standards.
25. Online merchants may be required by payment providers to use only PCI DSS-validated service providers.
26. Shopping cart software providers may be held liable by payment networks for non-compliance with security regulations.
27. Abandoned cart data is preserved and may be utilized for online marketing purposes.
28. The administration area may be accessed either through a web browser or a local desktop application.
29. The software does not support motorized shopping carts or physical wheelchairs.
30. The storefront's appearance can be modified to merge with the non-e-commerce pages of the website.
