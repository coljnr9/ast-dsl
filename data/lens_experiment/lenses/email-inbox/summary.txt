Email is a digital communication system designed to transmit and receive messages over computer networks using a store-and-forward model. The core purpose of the system is to allow users to exchange asynchronous messages without requiring both parties to be online simultaneously. The system relies on a standardized addressing format consisting of a local-part and a domain name (e.g., user@example.com). Messages are structured into a "content" section—comprising a header with metadata like the sender, recipient, and subject, and a body containing the actual message—and an "envelope" used by the underlying protocols for routing.

The operation of the system involves several specialized software agents and protocols. A sender uses a Mail User Agent (MUA) to compose a message, which is then submitted via Simple Mail Transfer Protocol (SMTP) to a Mail Submission Agent (MSA). The system then utilizes the Domain Name System (DNS) to identify the recipient’s Mail Transfer Agent (MTA) through MX records. The message may pass through multiple MTAs before a Mail Delivery Agent (MDA) places it into the recipient’s mailbox. Finally, the recipient retrieves the message from the server using protocols like POP3, which typically downloads and deletes the message from the server, or IMAP, which allows for synchronized management across multiple devices.

The system is governed by strict formatting rules and constraints defined by various RFC standards. Originally restricted to 7-bit ASCII text, the system now supports multimedia attachments and diverse character sets through Multipurpose Internet Mail Extensions (MIME). While delivery is generally reliable, it is not instantaneous; delays can occur due to server maintenance, large attachments, or greylisting by spam filters. As a critical security constraint, many modern servers refuse to accept mail from "open relays" to prevent the spread of unsolicited bulk email. If a message cannot be delivered after a standard retry period—usually up to five days—the system is designed to generate a "bounce message" to notify the original sender of the failure.
