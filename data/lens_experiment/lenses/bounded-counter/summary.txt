The CTUD is a bidirectional bistable function block designed to manage an internal numeric counter that can both increment and decrement based on specific input signals. Its primary purpose is to track a current value, known as CV, within the context of defined upper and lower limits. The system operates by monitoring the rising edges of two primary boolean triggers: the count-up input and the count-down input. When a rising edge is detected on the count-up signal, the current value increases by one; conversely, a rising edge on the count-down signal reduces the value by one. The system also monitors two status outputs: the upper-limit indicator becomes true whenever the current value is greater than or equal to a user-defined preset value, while the lower-limit indicator becomes true specifically when the current value reaches zero.

The behavior of the counter is governed by a hierarchy of control inputs that take precedence over the incremental counting logic. The reset signal serves as the highest priority override; when active, it forces the current value to zero regardless of other inputs. Similarly, the load signal allows the current value to be immediately synchronized with the preset value. The data types used for the preset and current values are words, which establishes the numerical range of the system. A critical constraint within the logic is that while the upper-limit signal triggers based on the relationship between the current value and the preset value, it does not inherently prevent the counter from exceeding that preset value if additional count-up triggers are received. This makes the system a flexible tracking mechanism rather than a hard-clamped limiter, requiring the calling program to manage state changes based on the boolean status outputs.
